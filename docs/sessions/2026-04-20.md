## 코드 변경 사항

### 신규 기능: KR/EN 이중 초안 시스템

- Claude가 한 번의 API 호출로 KR + EN 초안 동시 생성
- JSON 구조화 출력: `frame`, `kr`, `en`, `tags_kr`, `tags_en`
- 분석 프레임 6종: Mearsheimer / Thomas Risse / Hecker / Arendt / Victor Cha / James Scott
- THE GOLDEN RULE: 이론가 이름 직접 언급 금지, 팩트 대비로 독자 스스로 결론 도달
- 3단계 fallback 파싱 (JSON → 코드블록 제거 → 정규식)
- 모델: claude-sonnet-4-5 → **claude-haiku-4-5-20251001** (비용 80%+ 절감)

### DB 스키마 추가 (`002_add_draft_fields.sql`)

```sql
ALTER TABLE drafts
  ADD COLUMN draft_text_en TEXT,
  ADD COLUMN tags_kr  JSONB DEFAULT '[]',
  ADD COLUMN tags_en  JSONB DEFAULT '[]',
  ADD COLUMN frame    TEXT,
  ADD COLUMN slot     TEXT DEFAULT 'KR';
```

### 보안 강화

| 이슈 | 조치 |
|------|------|
| `/api/cron` 인증 없음 | CRON_SECRET Bearer 토큰 인증 추가 |
| `/api/reject` 인증 없음 | APPROVE_PASSWORD Bearer 토큰 인증 추가 |
| CORS 와일드카드 | NEXT_PUBLIC_APP_URL 기반 화이트리스트 |
| 쿠키 인증 버그 | `=== "authenticated"` → 존재 여부 체크로 수정 |
| JSONB 이중 인코딩 버그 | `json.dumps()` 제거, Python list 직접 전달 |
| Supabase Client 매번 재생성 | 싱글턴 패턴 적용 |
| KR 300자 vs 서버 280자 불일치 | 프롬프트 280자로 통일 |
| `vercel.json` comment 필드 오류 | comment 필드 제거 |
